Работа с типографикой в Android-проекте — это не только выбор красивого шрифта. Чтобы интерфейс выглядел профессионально, был доступным и не «тормозил», нужно учитывать несколько технических и эстетических аспектов.

Вот основные «столпы» уверенной работы с текстом:

---

## 1. Масштабируемость (Accessibility)

Пользователи могут менять размер шрифта в настройках системы (например, пожилые люди ставят «Огромный»).

* **Используйте `sp` (Scalable Pixels):** Никогда не используйте `dp` для текста. `sp` учитывает системные настройки масштаба.
* **Тестирование:** Всегда проверяйте, как выглядит ваш экран при максимальном увеличении шрифта в настройках Android. Текст не должен «налезать» друг на друга или обрезаться краями контейнера.

## 2. Вертикальный ритм и `lineHeight`

Стандартное межстрочное расстояние (интерлиньяж) часто слишком тесное для чтения длинных текстов.

* В Compose задавайте `lineHeight` в `TextStyle`. Хорошее правило: межстрочное расстояние должно быть примерно **1.2–1.5 раза больше**, чем размер шрифта (`fontSize`).
* Пример: `fontSize = 16.sp`, `lineHeight = 24.sp`.

## 3. Оптимизация ресурсов

Шрифты — это тяжелые бинарные файлы.

* **Форматы:** Android лучше всего работает с `.ttf` и `.otf`.
* **Subset (Подмножества):** Если вы используете декоративный шрифт только для цифр или заголовков, удалите из файла лишние символы (китайские иероглифы, расширенную латиницу) с помощью инструментов вроде *fonttools*. Это может уменьшить вес файла с 500 КБ до 30 КБ.

## 4. Иерархия через `Typography`

Не настраивайте каждый `Text` вручную. Создайте единую систему в `Type.kt`:

* **Headline:** Для крупных заголовков.
* **Title:** Для названий разделов.
* **Body:** Для основного текста (самый важный стиль для чтения).
* **Label/Caption:** Для подписей и мелких деталей.

Это позволит вам изменить шрифт во всем приложении, поменяв всего одну строчку в конфиге темы.

## 5. Работа с «прыжками» текста (Layout Invalidation)

В Compose изменение текста или его стиля может вызвать перерисовку (recomposition).

* Если вам нужно плавно менять размер шрифта (например, при анимации), лучше использовать `graphicsLayer { scaleX = ... }`, а не менять `fontSize` в каждом кадре, так как изменение размера шрифта — это тяжелая операция для процессора.

### Таблица-шпаргалка по выбору шрифтов:

| Параметр | Совет |
| --- | --- |
| **Количество шрифтов** | Не более 2-х разных семейств на проект (например, один для заголовков, один для чтения). |
| **Контраст** | Заголовки должны быть заметно жирнее или больше основного текста. |
| **Длина строки** | Оптимально для чтения — 45–75 символов в строке. На широких экранах планшетов ограничивайте ширину текстовых блоков. |

> Не забывайте про **Padding** вокруг текста. Текст «вплотную» к краям экрана выглядит дешево. Всегда оставляйте «воздух».
