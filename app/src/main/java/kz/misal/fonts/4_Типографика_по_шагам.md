Давайте создадим надежную систему типографики для вашего Compose-проекта. 

Мы настроим файл `Type.kt` так, чтобы он был гибким, поддерживал ваш новый рукописный шрифт и соответствовал стандартам Material Design 3.

Ниже представлена структура, которую можно просто скопировать и адаптировать.

---

## 1. Настройка FontFamily

Сначала объединим файлы шрифтов. Предположим, вы выбрали **Caveat** для акцентов и используете стандартный системный шрифт для основного чтения.

```kotlin
// ui/theme/Type.kt

val CaveatFontFamily = FontFamily(
    Font(R.font.caveat_regular, FontWeight.Normal),
    Font(R.font.caveat_bold, FontWeight.Bold)
)

// Можно оставить системный или добавить свой основной (например, Roboto)
val MainFontFamily = FontFamily.Default 

```

---

## 2. Создание объекта Typography

В Material 3 используется 15 стандартных стилей. Мы переопределим только основные, чтобы задать характер приложению.

```kotlin
val AppTypography = Typography(
    // Заголовки (используем рукописный шрифт)
    displayLarge = TextStyle(
        fontFamily = CaveatFontFamily,
        fontWeight = FontWeight.Bold,
        fontSize = 57.sp,
        lineHeight = 64.sp,
        letterSpacing = (-0.25).sp
    ),
    headlineMedium = TextStyle(
        fontFamily = CaveatFontFamily,
        fontWeight = FontWeight.SemiBold,
        fontSize = 28.sp,
        lineHeight = 36.sp
    ),

    // Основной текст (используем системный шрифт для лучшей читаемости)
    bodyLarge = TextStyle(
        fontFamily = MainFontFamily,
        fontWeight = FontWeight.Normal,
        fontSize = 16.sp,
        lineHeight = 24.sp,
        letterSpacing = 0.5.sp
    ),
    
    // Подписи и мелкий текст
    labelMedium = TextStyle(
        fontFamily = MainFontFamily,
        fontWeight = FontWeight.Medium,
        fontSize = 12.sp,
        lineHeight = 16.sp
    )
)

```

---

## 3. Подключение к теме

Чтобы эти стили заработали, их нужно передать в `MaterialTheme` в файле `Theme.kt`.

```kotlin
@Composable
fun MyProjectTheme(
    content: @Composable () -> Unit
) {
    MaterialTheme(
        colorScheme = lightColorScheme(), // или ваша цветовая схема
        typography = AppTypography,       // Подключаем нашу типографику
        content = content
    )
}

```

---

## Что это дает на практике?

1. **Чистый код в UI**: Вам больше не нужно писать `fontFamily = ...` в каждом компоненте.
2. **Легкое управление**: Если завтра вы решите, что `bodyLarge` должен быть чуть больше, вы измените это в одном месте.
3. **Контраст**: Использование рукописного шрифта **только** для `display` и `headline` стилей создаст отличный визуальный акцент, не жертвуя читаемостью основного контента.

### Маленький лайфхак:

Если вы хотите использовать рукописный шрифт «точечно» прямо внутри текста, можно создать расширение:

```kotlin
val TextStyle.handwritten: TextStyle
    get() = this.copy(fontFamily = CaveatFontFamily)

// Использование:
Text("Привет!", style = MaterialTheme.typography.bodyLarge.handwritten)

```
