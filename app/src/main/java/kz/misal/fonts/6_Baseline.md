Работа с **Baseline** (базовой линией) — это секрет профессиональной верстки. Если вы просто поместите два текста разного размера в один `Row`, они выровняются по верхнему краю или по центру, из-за чего строка будет выглядеть «прыгающей».

Вот как это работает и как этим управлять в Compose.

---

## Что такое Baseline?

**Baseline** — это воображаемая линия, на которой стоят все буквы без учета их «хвостов» (как у букв *у, д, р*). Когда мы читаем, наш глаз сканирует именно эту линию. Если у двух соседних слов она разная, мозг воспринимает это как ошибку или неаккуратность.

---

## 1. Выравнивание текстов разного размера

Чтобы большой заголовок и маленькая подпись в одном `Row` стояли на одной «полке», используйте модификатор `alignByBaseline()`.

**Плохо (по умолчанию):**

```kotlin
Row {
    Text("Заголовок", fontSize = 30.sp)
    Text("подзаголовок", fontSize = 14.sp) // Будет висеть вверху или по центру
}

```

**Хорошо:**

```kotlin
Row {
    Text(
        text = "Заголовок",
        fontSize = 30.sp,
        modifier = Modifier.alignByBaseline()
    )
    Text(
        text = "подзаголовок",
        fontSize = 14.sp,
        modifier = Modifier.alignByBaseline()
    )
}

```

*Результат: буквы обоих текстов стоят на одной невидимой линии, несмотря на разницу в размере.*

---

## 2. Выравнивание текста с иконкой

Иконки — это графические объекты, у них нет встроенного понятия «базовая линия». Если вы примените к иконке `alignByBaseline()`, она просто выровняется по своему нижнему краю, что часто выглядит некрасиво (иконка будет казаться слишком высокой).

Для этого в Compose есть специальный модификатор `alignBy(FirstBaseline)`.

```kotlin
Row(verticalAlignment = Alignment.CenterVertically) {
    Icon(
        imageVector = Icons.Default.Info,
        contentDescription = null,
        modifier = Modifier.size(24.dp).alignBy { it.measuredHeight } // Кастомное выравнивание
    )
    Text(
        text = "Важное сообщение",
        modifier = Modifier.alignByBaseline()
    )
}

```

**Практический совет:** Чаще всего для пары «Иконка + Текст» лучше всего работает стандартное `verticalAlignment = Alignment.CenterVertically` в самом `Row`. Но если текст очень крупный, а иконка маленькая, используйте `Modifier.padding(bottom = ...)` у иконки, чтобы визуально «подтянуть» её к базовой линии текста.

---

## 3. Вложенные Row и общая базовая линия

Если внутри вашего `Row` есть другие контейнеры (например, другой `Row` или `Column`), и вы хотите, чтобы всё выровнялось по тексту внутри них, используйте `alignByBaseline()` на самих контейнерах.

```kotlin
Row {
    // Весь этот блок будет "стоять" на базовой линии текста внутри него
    Row(modifier = Modifier.alignByBaseline()) {
        Icon(...)
        Text("Текст 1")
    }
    Text("Текст 2", modifier = Modifier.alignByBaseline())
}

```

---

## Резюме для работы:

1. **Текст + Текст:** Всегда используйте `Modifier.alignByBaseline()` для обоих элементов.
2. **Текст + Иконка:** Обычно хватает `Alignment.CenterVertically` в параметрах `Row`.
3. **Сложные элементы:** Используйте `alignBy(FirstBaseline)`, чтобы указать, по какой именно линии внутри сложного компонента нужно равняться.

Это особенно важно при использовании вашего рукописного шрифта, так как у декоративных шрифтов базовые линии могут визуально отличаться от стандартных.
